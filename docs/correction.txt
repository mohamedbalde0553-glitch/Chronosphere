la table `hr_employees`. La colonne réelle est probablement `is_active`.

1. Lance : php artisan tinker

2. Cherche tous les fichiers qui utilisent `status` sur hr_employees :
   - Contrôleurs Module 2
   - Models Employee
   - Vues Blade Module 2
3. Soit :
   a) Renomme `is_active` en `status` (avec migration) si tu juges 
      que `status` est plus pertinent
   b) Ou corrige le code pour utiliser `is_active` à la place
4. Choisis la solution la plus propre et applique-la partout
5. Teste : http://127.0.0.1:8000/shifts ne doit plus planter
6. Commit : "fix(hr): correction colonne status manquante hr_employees"
