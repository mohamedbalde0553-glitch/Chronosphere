PHASE J — EMPLOIS PÉRIODIQUES (NOUVEAU - Module 2 enrichi)
═══════════════════════════════════════════════════════════════

CONTEXTE : Le Module 2 doit gérer les HORAIRES PÉRIODIQUES par 
département ou section. Exemple métier réel :

🏢 Département "Production" :
   - Période : janvier à juin 2026
   - Horaires fixes : lundi-vendredi, 8h-16h
   - Tous les employés du département héritent automatiquement de 
     ces horaires sauf exception individuelle

🏢 Section "Accueil" (sous-département) :
   - Période : continue, sans fin
   - Horaires : 9h-18h avec rotation week-end
   - Heures supplémentaires automatiquement comptées le dimanche

C'est une fonctionnalité PRO qui apporte un vrai cas d'usage RH.

J.1 — Modélisation BDD
─────────────────────────────────────────────────────────────
Crée ces nouvelles tables (préfixe hr_) :

Table hr_work_schedules (modèles d'horaires) :
- id, name (ex: "Horaire Production Hiver"), description
- start_date, end_date (période de validité, end_date nullable)
- department_id (FK, nullable si concerne plusieurs)
- section_id (FK, nullable)
- created_by, timestamps, soft deletes

Table hr_work_schedule_days (jours et heures) :
- id, work_schedule_id (FK)
- day_of_week (0=dimanche, 6=samedi)
- start_time (TIME), end_time (TIME)
- break_minutes (INT)
- is_overtime_eligible (BOOLEAN)
- multiplier (DECIMAL pour heures sup/nuit/week-end)

Table hr_employee_schedule_overrides (exceptions individuelles) :
- id, employee_id, work_schedule_id (le schedule de référence)
- override_start_date, override_end_date
- reason (TEXT)
- timestamps

Migration + Models + Relations Eloquent

J.2 — Logique métier (Services)
─────────────────────────────────────────────────────────────
Crée des Services :

WorkScheduleService :
- generateShiftsFromSchedule($scheduleId, $startDate, $endDate) :
  génère automatiquement les shifts pour TOUS les employés concernés
- calculateExpectedHours($employeeId, $startDate, $endDate) :
  retourne les heures théoriques attendues sur la période
- detectConflicts($scheduleId) : vérifie qu'il n'y a pas de 
  chevauchement avec d'autres schedules actifs
